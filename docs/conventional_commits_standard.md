# Conventional Commits Standard 
## Einer Konvention für einheitliche, maschinenlesbare Commit-Messages

Das Format ist immer: `typ`: `beschreibung`

Die häufigsten Typen:

| Tag | Bedeutung |
| :--- | :--- |
| feat | Neues Feature |
| fix | Bugfix | 
| docs | Nur Dokumentation geändert | 
| chore | Wartungsarbeiten — kein Feature, kein Fix, kein ProduktionscoderefactorCode umgebaut, Verhalten unverändert |
| test | Tests hinzugefügt oder geändertstyleFormatierung, Whitespace (kein Logik-Änderung) |
